## 三、Agent → 模型推荐配置

基于各 Agent 的定位和任务类型，推荐以下配置：

| Agent | 定位 | 推荐模型 | 备选模型 | 理由 |
|-------|------|---------|---------|------|
| **Claude Code** | 主力编程 Agent，复杂工程任务 | **MiniMax-M2.5** (SWE-Bench 80.2%) | GLM-5.1 / qwen3.6-plus | 编程能力最强，工具调用和搜索 SOTA |
| **Hermes** | 通用 Agent，多用途编排 | **qwen3.6-plus** (1M 上下文 + thinking) | GLM-5 / mimo-v2.5-pro | 1M 上下文适合持有完整项目上下文，thinking 模式适合编排决策 |
| **Codex** | 代码审查 / 质量保障 | **GLM-5** (745B/44B) | glm-4.7 / kimi-k2.5 | 开源 SOTA 编程 + Agent 能力，复杂审查可靠 |
| **OpenCode** | 快速编码 / 日常开发 | **GLM-4.7** (355B/32B) | Qwen3.5-35B-A3B / mimo-v2.5 | 编程强且稳，响应速度快，性价比高 |
| **Pi-coding-agent** | 轻量/高频编码 | **Copaw-9B** (本地) → **mimo-v2-flash** | Qwen3.5-35B-A3B | 零成本本地优先，fallback 到最便宜云端 |

### 补充场景推荐

| 场景 | 推荐模型 | 理由 |
|------|---------|------|
| **视觉编程 (截图/UI → 代码)** | kimi-k2.5 | 唯一支持视觉编程的开源 SOTA 模型 |
| **超长文档/全仓库分析** | mimo-v2.5-pro / qwen3.6-plus | 1M 上下文 |
| **边缘/离线部署** | Qwen3.5-35B-A3B (3B 激活) | 24GB 显存即可运行 |
| **多模态理解** | kimi-k2.5 / Qwen3.5-35B-A3B / mimo-v2.5-pro | 原生多模态 |
| **高频低成本** | mimo-v2-flash (¥0.07/M tokens 缓存命中) | 成本最优 |
| **8h+ 长程自主任务** | GLM-5.1 | 官方支持最长 8 小时自主工作 |
