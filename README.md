# One Piece

> 企业级 Agent 协作系统 — 多 Agent，一个整体

## 命名寓意

致敬同名动漫《One Piece》。路飞作为船长，带领各有所长的伙伴（航海士、厨师、剑士、医生……）在大海上航行。外人看到的不是分散的个体，而是一个完整的海贼团。

**One Piece 直译为"一个整体"**，寓意多 Agent 协作系统在用户视角是一个统一实体——内部复杂的任务分解、委托执行、人工验证由系统自动完成，用户只需与"船长"对话，无需关心每个"船员"如何工作。

## 解决什么问题

当前多 Agent 协作模式像一场"传话游戏"：

```
错误模式（信息逐步失真）：
Agent A → 输出摘要 → Agent B → 输出摘要 → Agent C
        上下文丢失        偏差累积        错误扩散
```

三大核心痛点：

- **上下文丢失**：每个 Agent 只接收到上一节点的输出摘要，缺乏完整上下文
- **理解偏差累积**：每个节点的解读偏差在流转中被不断放大
- **错误扩散**：前置节点的错误被后续节点默认接受并放大，修复成本极高

## 我们的方案

**同一个上下文 → 同一个主 Agent 持有**

主 Agent 像船长一样统筹全局：

```
正确模式（委托 sub-agent）：
Agent-A (主 Agent，持有完整上下文)
├── Sub-Agent-A1 (处理子任务 1)
├── Sub-Agent-A2 (处理子任务 2)
└── Sub-Agent-A3 (处理子任务 3)
         ↓
   关键节点请求人类验证
         ↓
   主 Agent 整合结果，统一交付
```

- **主 Agent 持有完整上下文**，负责统筹协调
- **sub-agent 仅处理原子化子任务**，完成后返回结果给主 Agent
- **主 Agent 在关键节点请求人类验证**，而非每个 sub-agent 都找人类
- **用户只与船长对话**，不需要关心水手如何工作

## 核心特性

### Human-in-the-Loop 智能验证

- **三级验证点**：前置验证（确认方向）、中间验证（关键决策）、最终验证（输出审核）
- **智能路由**：基于任务类型、员工专业领域、历史反馈质量自动推荐最合适的验证者
- **错误阻断**：每个验证点都是"断路器"，支持通过/退回/转派，错误不会扩散到下游
- **强制验证**：文件修改、命令执行、外部系统调用、数据删除等关键操作前强制人工确认

### 三层 Skill 体系

```
atoms 原子层（砖块） → molecules 分子层（工件） → compounds 复合层（工程）
```

| 层级            | 定位     | 特征                               | 示例               |
| ------------- | ------ | -------------------------------- | ---------------- |
| atoms 原子层     | 最小可执行块 | 高确定性、低抽象、不调别的 Skill              | 读取文件、执行命令、调用 API |
| molecules 分子层 | 有界任务组合 | 2-10 个 atoms 链式拼接，组合逻辑写在 Skill 里 | 代码审查、安全扫描、测试执行   |
| compounds 复合层 | 复杂工作流  | 多 molecules 协作，真正交出判断，需要人在驾驶座    | 功能开发、事故排查        |

> **原子要稳 · 分子要可靠 · 复合层才能放心交出判断**

### 多协作模式

- **主从架构（推荐）**：Lead 持有完整上下文，sub-agent 处理专项，统一汇报
- **并行验证（竞争假设）**：多个 sub-agent 独立调查，结果汇总后人工决策
- **多维度审核**：安全/性能/架构并行审核，避免单一视角盲区

### Skill 沉淀与发现

- 反思引擎定期分析历史任务，识别重复的 atoms/molecules 调用模式
- 自动提取高价值的 Skill 组合，生成 Skill 候选
- 员工审核后启用，新任务自动匹配推荐

## 技术栈

| 层级        | 技术                                | 说明                         |
| --------- | --------------------------------- | -------------------------- |
| 业务服务      | Next.js (App Router) + TypeScript | 任务管理、审核、Skill 管理           |
| 核心服务      | Go                                | Agent 调度、Approval Gateway  |
| 数据库       | PostgreSQL                        | 关系型数据存储                    |
| 缓存/队列     | Redis                             | 缓存、会话、任务队列                 |
| 实时通信      | WebSocket                         | 任务状态同步、审批通知                |
| Agent 运行时 | 可插拔 Provider                      | Claude Code、Hermes、Codex 等 |
